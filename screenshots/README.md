# Screenshots Instructions

This directory should contain screenshots of the PrepKit application.

## Required Screenshots

Please capture the following screens and save them with these exact filenames:

1. **dashboard.png**
   - Navigate to: `/dashboard`
   - Show: ProgressCircle, MacroBars, Quick Actions, Today's Meals
   - Recommended size: 1200x800px

2. **recipe-list.png**
   - Navigate to: `/recipes`
   - Show: Search bar, Category chips, 2-column grid
   - Make sure some category is selected to show the filter
   - Recommended size: 1200x800px

3. **recipe-detail.png**
   - Navigate to: `/recipes/[any-recipe-id]`
   - Show: Hero image, tabs, nutrition info
   - Recommended size: 1200x1600px (vertical scroll)

4. **meal-planner.png**
   - Navigate to: `/planner`
   - Show: Day selector, meal type sections with some meals added
   - Add at least 2-3 meals to demonstrate the UI
   - Recommended size: 1200x1600px (vertical scroll)

5. **shopping-list.png**
   - Navigate to: `/shopping-list`
   - Show: Progress bar, some checked items, some unchecked
   - Make sure to check a few items to show the animation
   - Recommended size: 1200x1200px

6. **profile.png**
   - Navigate to: `/profile`
   - Show: User avatar, daily goals grid, settings
   - Recommended size: 1200x800px

## How to Capture

### Using Browser DevTools (Chrome/Edge)
1. Open DevTools (F12)
2. Enable Device Toolbar (Ctrl+Shift+M)
3. Set device to "Responsive"
4. Set width to 390px or 414px (iPhone size)
5. Take screenshot using DevTools screenshot feature

### Using Browser Extension
- Use "GoFullPage" or similar extension
- Capture full page for vertical screens
- Save as PNG

### Using Screenshot Tools
- Windows: Snipping Tool or Snip & Sketch
- macOS: Cmd+Shift+4
- Linux: gnome-screenshot or flameshot

## Tips
- Use dark theme screenshots
- Make sure UI is fully loaded (no skeleton loaders)
- Include realistic data (not empty states)
- Ensure text is readable
- Use consistent viewport size

After capturing all screenshots, they will automatically appear in the README.md!
